# 무한매수법 매수/매도 수량 계산 대시보드 구현 계획

## Context (배경)

`CLAUDE.md`에 정의된 **무한매수법**(TQQQ 대상 분할 매수 전략)의 규칙에 따라, 매일
매수/매도해야 할 **주문 가격과 수량을 자동 계산**해 주는 대시보드가 필요하다.
현재 저장소는 `CLAUDE.md`, `README.md`만 있는 그린필드 상태다.

**목표 결과물**: 안드로이드 등 모바일 브라우저에서 바로 열 수 있는 웹 기반
대시보드. SEED·평단·수량 등 상태는 브라우저에 저장되고, TQQQ 현재가는
실시간 API로 자동 조회되며, 오늘 넣어야 할 LOC 매수/매도 주문이 즉시 계산된다.

### 확정된 방향 (사용자 결정)
- **형태**: 웹 기반 정적 앱 (안드로이드 모바일 브라우저에서 실행) — 모바일 우선 반응형
- **가격 소스**: Finnhub 무료 API (실시간 미국주식 시세, 브라우저 CORS 지원, 무료 API 키 필요)
- **상태 저장**: 브라우저 localStorage
- **기술 스택**: 빌드 도구 없는 단일 페이지 정적 웹앱 (HTML/CSS/바닐라 JS) — GitHub Pages 배포 가능

## 계산 로직 (CLAUDE.md 규칙 → 공식)

모든 계산의 핵심. 구현의 정확성을 위해 먼저 명세를 확정한다.

### 입력값
| 항목 | 출처 |
|---|---|
| SEED (총 예산) | 사용자 입력 |
| 분할수 (40/30/20) | 사용자 선택 |
| 목표수익률 % | 사용자 입력 |
| 평단 (평균단가) | 사용자 입력 |
| 수량 (보유 주식 수) | 사용자 입력 |
| 전일 종가 / 현재가 | Finnhub API 자동 (수동 override 가능) |

### 파생 계산
- `일일매수금액 = SEED / 분할수`
- `총매수금액 = 평단 × 수량`
- `진행률 = 총매수금액 / SEED`  (0~1 비율)
- `매도점 = 평단 × (1 + 목표수익률/100)`
- `별퍼센트(%) = 목표수익률 − (2 × 진행률) × 목표수익률 = 목표수익률 × (1 − 2×진행률)`
- `별퍼센트 매도점 = 평단 × (1 + 별퍼센트/100)`
- `남은 예산 = SEED − 총매수금액`
- `회차(진행일수) ≈ 총매수금액 / 일일매수금액`

### 오늘의 주문 산출

**첫날 (수량 == 0, 규칙 6)**
- 기준가 = `전일종가 × 1.10`
- LOC 매수: 가격 = 기준가, 수량 = `floor(일일매수금액 / 기준가)`

**일반일 매수 (규칙 1·2) — 일일매수금액을 절반씩 양방향 LOC 매수**
- 매수①: LOC BUY, 가격 = `평단`, 수량 = `floor((일일매수금액/2) / 평단)`
- 매수②: LOC BUY, 가격 = `별퍼센트 매도점`, 수량 = `floor((일일매수금액/2) / 별퍼센트매도점)`
  - (종가가 각 지정가보다 낮을 때 체결되는 LOC 주문)

**일반일 매도 (규칙 3·4) — 보유수량 1/4 + 3/4 분할**
- 매도①(별도매도): LOC SELL, 가격 = `별퍼센트 매도점`, 수량 = `round(수량 × 1/4)`
- 매도②(전량목표): 지정가 SELL, 가격 = `매도점`, 수량 = `수량 − 매도①수량` (약 3/4)

**강제매도 경고 (규칙 5)**
- `진행률 ≥ 1` (SEED 소진)인데 매도점 미도달 시:
  경고 배너 + 종가 강제매도 수량 = `round(수량 × 1/4)` 표시

> 수량은 정수 주(주식은 1주 단위)로 계산하며 예산 초과 방지를 위해 `floor` 적용
> (매도 수량은 `round`). 이 반올림/내림 규칙은 구현 시 상수로 명시.

## 구현 파일 구조

빌드 도구 없이 정적 파일로 구성 (GitHub Pages 바로 배포 가능):

```
index.html      # 대시보드 마크업 + 뷰포트 메타(모바일), 섹션 골격
styles.css      # 모바일 우선 반응형 스타일, 카드/표 레이아웃, 다크 대응
app.js          # 상태관리 + 계산엔진 + 렌더링 (진입점)
calc.js         # 순수 계산 함수 (위 공식) — 테스트 가능하게 분리
price.js        # Finnhub 가격 조회 모듈 (provider 추상화)
storage.js      # localStorage 로드/저장 유틸
README.md       # 사용법 + Finnhub API 키 발급/설정 + 배포 안내 (갱신)
```

> 단일 파일 선호 시 `calc/price/storage`를 `app.js`에 인라인 가능하나,
> 계산 로직 검증 편의를 위해 `calc.js` 분리를 권장.

### 각 파일 역할
- **calc.js**: 위 "계산 로직"을 순수 함수로 구현
  (`computeDerived(inputs)`, `computeOrders(inputs, price)` → 주문 객체 배열 반환).
  외부 의존성 0 → 단위 테스트/재사용 용이.
- **price.js**: `fetchPrice(symbol='TQQQ')` — Finnhub `GET /api/v1/quote?symbol=TQQQ&token=KEY`
  호출, `{current, prevClose}` 반환. 키는 storage에서 읽고, 실패 시 수동 입력값으로 폴백.
- **storage.js**: 입력 상태 + API 키를 `localStorage`에 저장/복원 (`stock.ib.*` 네임스페이스).
- **app.js**: 입력 폼 이벤트 → 상태 갱신 → calc 호출 → DOM 렌더. 새로고침(가격) 버튼.

### UI 섹션 (index.html)
1. **설정 입력**: SEED, 분할수(select 40/30/20), 목표수익률, 평단, 수량, Finnhub API 키.
2. **가격 카드**: TQQQ 현재가/전일종가 + [새로고침] 버튼 + 수동 override 필드.
3. **요약 카드**: 일일매수금액, 총매수금액, 진행률%, 회차, 매도점, 별퍼센트%,
   별퍼센트 매도점, 남은 예산.
4. **오늘의 주문**:
   - 매수 표 (첫날 1행 또는 일반일 2행: 유형/가격/수량/금액)
   - 매도 표 (LOC 1/4 + 지정가 3/4)
   - 강제매도 경고 배너 (조건부)

## Finnhub 연동 세부

- 엔드포인트: `https://finnhub.io/api/v1/quote?symbol=TQQQ&token={API_KEY}`
- 응답: `{ c: 현재가, pc: 전일종가, ... }` → `current=c`, `prevClose=pc`
- API 키는 앱 내 설정 필드에서 입력받아 localStorage 저장 (코드/저장소에 하드코딩 금지).
- CORS: Finnhub 무료 티어는 브라우저 요청 지원. 실패/한도초과 시 수동 가격 입력으로 자동 폴백.
- README에 무료 키 발급 절차 명시.

## 검증 방법 (end-to-end)

1. **계산 검증**: `calc.js`에 대해 대표 시나리오 수동 검산
   - 예) SEED=4000, 분할수=40 → 일일매수금액=100. 평단=50, 수량=40 →
     총매수금액=2000, 진행률=0.5 → 별퍼센트=0(목표10% 기준), 매도점=55 등
     기대값과 화면 출력 대조.
   - 첫날(수량0), 진행률≥1(강제매도 경고) 경계 케이스 확인.
   - (선택) 간단한 브라우저 콘솔 테스트 또는 `console.assert` 스니펫으로 공식 검증.
2. **가격 연동**: 유효한 Finnhub 키 입력 후 [새로고침] → TQQQ 현재가/전일종가 표시 확인.
   키 없음/오류 시 수동 입력 폴백 동작 확인.
3. **저장 검증**: 값 입력 → 새로고침(F5) → localStorage 복원 확인.
4. **모바일 확인**: 개발자도구 모바일 뷰 또는 실제 안드로이드 브라우저에서 반응형 레이아웃 확인.
5. **배포(선택)**: GitHub Pages로 `claude/infinite-buying-dashboard-plan-ujr8ex`
   또는 기본 브랜치 게시 후 모바일 접속 테스트.
